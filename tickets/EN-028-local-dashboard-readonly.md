# EN-028 Local Dashboard Read-only

## Goal

Add a minimal local read-only dashboard to inspect candidate posts generated by
`scripts\x_fetch_accounts_probe.py`.

## Scope

- Read `runtime/outputs/x_candidates.json` by default.
- Render summary counts and candidate rows.
- Support filters for account, media, new candidates, and minimum score.
- Keep the dashboard read-only with no approval, editing, publishing, scheduler,
  auth, or database behavior.
- Avoid frontend build tooling and network dependencies.

## Validation

- `python -m compileall app tests scripts`
- `python -m pytest -p no:cacheprovider --basetemp=runtime/pytest-tmp`
- `git status --short`
- `git ls-files runtime "*/x_headers.json" x_headers.json .env ".env.*" "*.db"`
